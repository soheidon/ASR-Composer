# ASR Composer

音声・動画ファイルから高精度な文字起こしを作成するデスクトップアプリケーションです。

## 概要

ASR Composerは、複数のASR（自動音声認識）エンジンをGUIから操作し、話者分離・多形式出力・中止機能を備えた文字起こしツールです。ローカル環境（Docker + GPU）で完結するため、音声データを外部に送信する必要がありません。

## 主な機能

### 文字起こし

- ドラッグ＆ドロップまたはファイル選択ダイアログで音声ファイルを読み込み
- 3種類のローカルASRエンジンから選択して文字起こしを実行
- 文字起こし実行中に中止ボタンで処理を中断可能
- 対応フォーマット: MP3, WAV, MP4, M4A, FLAC

### ローカルASRエンジン

| エンジン | モデル | 特徴 |
|---|---|---|
| ReazonSpeech | ESPnet v3.0.0 | 日本語特化、高精度 |
| Kotoba Whisper | v2.2 | 日本語特化、高速 |
| Qwen3 ASR | Qwen3-ASR-1.7B | 多言語対応（30言語） |

### 出力形式

文字起こし結果を以下の形式で保存できます。チェックボックスで必要な形式を選択してください。

| 形式 | 拡張子 | 用途 |
|---|---|---|
| テキスト | `.txt` | プレーンテキスト、話者ラベル付き |
| JSON | `.json` | 構造化データ、プログラム連携 |
| Markdown | `.md` | 文書作成、GitHub等 |
| SRT | `.srt` | 字幕ファイル、動画編集ソフト対応 |
| CSV | `.csv` | 表計算ソフト対応（UTF-8 BOM付き） |
| VTT | `.vtt` | WebVTT字幕、HTML5動画対応 |

### 話者分離

- pyannote speaker-diarizationによる自動話者分離
- 話者数の自動検出または手動指定（1〜8人）
- 話者分離のON/OFF切替

### 言語選択

- エンジンごとに最適な言語候補を表示
- Qwen3 ASRは30言語に対応
- 自動検出もサポート

### その他の機能

- 保存先フォルダの指定（未指定時はDownloadsフォルダ）
- 保存先パスの記憶（次回起動時に復元）
- 文字起こし結果のコピー・手動保存
- APIプロバイダーの設定と接続テスト
- 補完LLM（GPT-4, Claude, Gemini等）の設定

## 動作環境

- **OS**: Windows 10/11（64ビット）
- **GPU**: NVIDIA GPU（CUDA 12.6対応推奨）
- **Docker**: Docker Desktop（ローカルASRエンジン使用時）
- **ディスク容量**: 各エンジンのDockerイメージで約10〜20GB

## インストール

### setup.exeによるインストール

1. [リリースページ](https://github.com/soheidon/ASR-Composer/releases)から最新の `ASR-Composer_0.1.0_x64-setup.exe` をダウンロード
2. setup.exeを実行してインストール
3. スタートメニューまたはデスクトップからASR Composerを起動

### ローカルASRエンジンのセットアップ

1. Docker Desktopをインストールし、起動状態にする
2. ASR Composerを起動し、設定画面の「Docker設定」を開く
3. 使用するエンジンの「インストール」ボタンを押してDockerイメージをビルド
4. インストール完了後、文字起こしページでエンジンを選択して使用

## 使い方

1. **音声ファイルを選択**: ドロップゾーンにファイルをドラッグするか、「ファイルを選択」ボタンをクリック
2. **ASRエンジンを選択**: API（クラウド）またはローカルを選択し、エンジンを指定
3. **出力設定**: 保存先と出力形式（TXT, SRT等）を選択
4. **文字起こしを開始**: 「文字起こしを開始」ボタンをクリック
5. **中止する場合**: 実行中に「中止」ボタンで処理を中断可能
6. **結果を確認**: 文字起こし結果が画面に表示され、選択した形式で保存先に書き出されます

## 開発

### ビルド要件

- Node.js 18以上
- Rust 1.77以上
- Docker Desktop

### 開手順

```bash
# 依存関係のインストール
npm install

# 開発サーバー起動
npm run tauri dev

# リリースビルド
npm run tauri build
```

### テスト

```bash
# TypeScriptテスト
npx vitest run

# Rustテスト
cd src-tauri && cargo test
```

# ASR Composer GUI仕様書

## 1. アプリ概要

### 1.1 アプリ名

**ASR Composer**

### 1.2 副題

**文字起こし作成・修正・統合**

### 1.3 目的

ASR Composerは、音声・動画ファイルから文字起こしを作成し、辞書・背景情報・LLMを用いて修正したうえで、分析や保存に使用できる正本を作成するデスクトップアプリである。

また、複数のASRエンジンで生成した文字起こし結果を比較・統合し、より正確な正本を作成する機能を持つ。

### 1.4 対象ユーザー

* 研究者
* インタビュー調査実施者
* 医療・臨床研究従事者
* 会議や講演の文字起こしを行うユーザー
* 複数ASRエンジンを比較・統合したいユーザー

### 1.5 技術構成

* フロントエンド：TypeScript + Vite
* デスクトップ基盤：Tauri v2
* バックエンド：Rust
* ウィンドウ：カスタムタイトルバー（`decorations: false`）
* ローカルLLM実行時：Pythonヘルパー
* 話者分離実行時：Docker上のpyannote
* アイコン：Google Material Symbols Outlined
* LLM：

  * ローカルLLM（Ollama等）
  * 外部API
* ASR：

  * ローカルASR
  * 外部API
  * 複数エンジン対応

# 2. デザイン方針

## 2.1 全体コンセプト

研究・業務用途に適した、落ち着いたデスクトップアプリとする。

派手な演出は避け、以下を重視する。

* 現在の処理段階が明確に分かる
* 入力ファイルと出力結果の関係が分かる
* ローカル処理とAPI処理を区別できる
* 長時間処理中でも進行状況が分かる
* 誤操作によるデータ消失を防ぐ
* 専門的な設定は必要な場合のみ表示する

## 2.2 推奨ビジュアル

* 基調：白または薄いグレー
* サイドバー：濃いグレーまたは淡い青灰色
* アクセント：落ち着いた青
* 成功：緑
* 警告：黄または橙
* エラー：赤
* ローカル処理：青系ラベル
* API処理：紫系ラベル
* Docker処理：グレーまたはコンテナアイコン

## 2.3 画面密度

* デスクトップ向け
* 最小推奨幅：1280px
* 標準幅：1440px以上
* 左右分割レイアウトを多用する
* 長文編集領域は十分な高さを確保する
* スマートフォン対応は不要

# 3. 全体レイアウト

## 3.1 アプリフレーム

Tauri v2 デスクトップアプリ。ウィンドウのデコレーションを無効（`decorations: false`）とし、カスタムタイトルバーを使用する。

画面は以下の2領域で構成する。

1. 上部タイトルバー（カスタムヘッダー）
2. メインコンテンツ

サイドバーは現在の実装には含まれない。

## 3.2 上部タイトルバー（カスタムヘッダー）

高さ：48px。3カラムCSS Gridレイアウトを使用する。

```
grid-template-columns: max-content minmax(32px, 1fr) max-content;
```

### カラム1（左側）：ブランド＋ナビゲーション

* ブランド名「ASR Composer」（太字、左パディング16px）
* 縦線セパレーター（幅1px、高さ20px、左右マージン16px）
* ナビゲーションリンク

  * 文字起こし（アクティブ状態：プライマリ色、下線3px）
  * 統合
  * 設定

### カラム2（中央）：ドラッグエリア

* 空のドラッグ可能領域（min-width: 32px）
* `mousedown`でTauri `startDragging()`を呼び出し、ウィンドウ移動を可能にする
* ダブルクリックで最大化トグル
* `data-tauri-drag-region`は使用しない（ボタンクリックとの競合を回避するため）

### カラム3（右側）：ウィンドウ操作ボタン

3つのボタン（各46px幅）：

* 最小化（アイコン：`remove`）
* 最大化／元に戻す（アイコン：`crop_square`／`filter_none`）
* 閉じる（アイコン：`close`、ホバー時にエラー背景色）

各ボタンはTauri API（`win.minimize()`、`win.toggleMaximize()`、`win.close()`）を`async/await`で呼び出す。

最大化状態の変化は`win.onResized()`イベントで監視し、アイコンを切り替える。

### Tauri権限（capabilities/default.json）

```json
[
  "core:default",
  "core:event:allow-listen",
  "core:event:allow-unlisten",
  "core:window:allow-minimize",
  "core:window:allow-toggle-maximize",
  "core:window:allow-is-maximized",
  "core:window:allow-close",
  "core:window:allow-start-dragging"
]
```

## 3.3 メインコンテンツ

中央揃え、max-width: 48rem。上下左右パディング24px、上部追加パディング20px。

セクションカードを縦並び（gap: 16px）で配置する。

## 3.4 tauri.conf.json ウィンドウ設定

```json
{
  "label": "main",
  "title": "ASR Composer",
  "width": 1440,
  "height": 900,
  "minWidth": 1280,
  "minHeight": 800,
  "decorations": false,
  "resizable": true,
  "maximizable": true,
  "minimizable": true,
  "closable": true
}
```

# 4. タブ1：文字起こし作成

**文字起こし作成**

## 4.2 目的

音声または動画ファイルを読み込み、ASRを実行し、辞書・背景情報・LLMで修正して正本を作成する。

## 4.4 ステップ1：入力ファイル

3カラム横向きレイアウト。

```
[アイコン] [テキスト群] [ボタン]
```

ドロップゾーンのスタイル：

* border: 2px dashed outline-variant
* border-radius: 12px
* padding: 20px 32px
* gap: 24px
* ホバー時：border-color primary-container、背景微変化

### 4.5 AIモデルと出力言語の選択

各行のレイアウト：flexbox、align-items: center、gap: 16px。

#### ASRエンジン行

* ラベル（80px固定幅）：「ASRエンジン」
* セレクトボックス（flex: 1）：Whisper Large v3 / Faster Whisper / Google Speech API
* 設定アイコンボタン（42px固定幅）：`tune`アイコン

#### 補完LLM行

* ラベル（80px固定幅）：「補完LLM」
* セレクトボックス（flex: 1）：なし / GPT-4 / Claude / Gemini
* 設定アイコンボタン（42px固定幅）：`tune`アイコン

#### 音声の言語行

* ラベル（80px固定幅）：「音声の言語」
* セレクトボックス（flex: 1）：自動検出 / 日本語 / 英語 / 中国語 / 韓国語
* プレースホルダー（42px固定幅、visibility: hidden、高さ揃え用）

## 4.6 出力設定

セクションカード内に2項目を縦並び（gap: 24px）で配置する。

* JSON
* Markdown
* SRT
* CSV
* VTT

* 発言内容を要約しない
